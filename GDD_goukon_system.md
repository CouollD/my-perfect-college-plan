## 設計思想

**グノーシアの「会議」を合コンに置き換える**

| グノーシア | 合コン乱戦 |
|---|---|
| 全員が自律的に発言する | 友達と相手女性が自律的に絡み合う |
| 誰が敵かわからない | 誰が誰を気に入ってるかわからない |
| コマンドの効果が状況で変わる | コマンドの効果が相手の属性と場の空気で変わる |
| 失敗すると信頼を失う | 失敗すると場の空気が悪くなる |
| プレイヤーは流れを読んで介入する | プレイヤーはどこに割り込むかを選ぶ |

**核心的な楽しさ：**
「けんたがゆなを取ってしまう前に、どう動くか」

## 3つの管理軸

### ① 場の空気ゲージ（0〜100）
- 全員に影響する。低いと好感度が上がりにくい。
- 盛り上がるイベントで上昇、気まずいイベントで下降
- 60以上：通常、40以下：コマンド効果半減、20以下：合コン崩壊

### ② 各女性の注目度（誰に注目が集まってるか）
- 注目が集まっている子にアプローチすると効果大
- 友達が注目を独占することがある
- 「まだ動くな」で誰の注目度が高いかわかる

### ③ 友達の暴走度
- コマンドを使わず放置すると友達が自律行動する
- 友達ごとに放置時の行動パターンが違う

| 友達 | 放置時の行動 | 結果 |
|---|---|---|
| 藤原けんた | 特定の女性に入れ込む | 独占フラグが立ちやすい |
| 上田ともや | ノリで暴走し始める | 場が盛り上がるが制御不能に |
| 石田こうへい | 静かに観察し続ける | 悪くはないが存在感が薄い |
| 佐藤ひろき | マニアックな話を始める | 場が静まる |
| 田中まさと | 段取りを整え始める | 二次会の流れが勝手に作られる |

## ターン構造

### 1ターンの流れ

```
① 自動イベント発生（ナレーション形式）
   例：「けんたがゆなに話しかけた。ゆなが笑った。」
   → ゆなの注目度↑、けんたの好感度(ゆな)↑

② 状況の変化が表示される
   「ゆなの注目がけんたに集まっている（暴走度2/5）」

③ プレイヤーの選択（AP1消費）
   A. 介入する → コマンド選択画面へ
   B. 静観する → 友達の自律行動が強まる
   C. 別の子へ → ターゲット変更

④ コマンド実行 → 結果テキスト表示
   例：「レシーブは任せた！→けんたの話をうまく受けて絡んだ。
       ゆなの視線が一瞬こちらに向いた。」

⑤ 次のターンへ
```

### フェーズ構成（5フェーズ × 2〜3ターン）

```
PHASE 1：乾杯（場の空気を作る）
PHASE 2：自己紹介・会話（誰が誰を狙うか見えてくる）
PHASE 3：ゲーム or 盛り上がり（友達の暴走が起きやすい）
PHASE 4：二次会の空気（独占フラグの解決or諦め）
PHASE 5：お開き（最後の勝負）
```

## 独占フラグシステム

友達が特定の女性に**3ターン以上関わる**と「独占フラグ」が立つ。

```
独占フラグが立つと：
・その女性へのプレイヤーの介入効果が半減
・「頼む、割り込んでくれ」コマンドが必要になる
・割り込みに失敗すると友達の好感度（プレイヤーへの）が下がる
・割り込みに成功するとその女性の注目が分散する
```

**「けんたを止めるか / ゆなを諦めるか」の判断がここで生まれる。**

## 情報の不完全さ

プレイヤーが最初からわかること：
- 相手女性の名前と雰囲気のみ
- 友達のパラメータ（自分が把握している範囲）

プレイヤーがわからないこと：
- 誰が誰を気に入っているか
- 友達の暴走度（見えにくい）
- 相手女性の好みの傾向（ヒントから推測する）

コマンドで情報を得る：
- **まだ動くな** → 誰の注目度が高いかわかる
- **俺のデータによると…（こうへい）** → 相手女性の傾向ヒントが1つ開示。「これはこうへいの観察データ」と明示する

## 自動イベントテキスト（例）

### PHASE 1（乾杯）
- 「けんたが乾杯の音頭を取った。場の空気が一気に明るくなった。」→ 空気+10
- 「ひろきが飲み物を間違えて気まずい空気が流れた。」→ 空気-8
- 「りょうが無言で座っているだけで、女子が視線を送っている。」→ りょうの注目度↑

### PHASE 2（会話）
- 「ゆなが『みんな何してるの？』と話を振った。けんたが即座に反応した。」→ けんた暴走度+1
- 「あおいが静かに○○に話しかけた。」→ あおいの注目度↑（プレイヤーへ）
- 「まさとがさりげなく追加注文を取った。誰かが助かった顔をした。」→ 空気+5

### PHASE 3（ゲーム）
- 「けんたが率先してゲームを仕切り始めた。独占フラグ2/5。」
- 「ともやが暴走し始めた。制御が難しい。」→ 空気+15 but ともや暴走度MAX

### PHASE 4（二次会）
- 「けんたがゆなに『二次会どうする？』と囁いた。独占フラグ成立。」
- 「めいが急に『ここって霊感ある人います？』と言い始めた。」→ 空気-15

## 未解決の設計問題

1. **1ターンの長さ** — テキストが多すぎるとテンポが悪くなる。1イベント＝2〜3行が上限か。
2. **コマンドの数** — 毎ターン全コマンドが出ると選択肢が多すぎる。状況に応じて2〜4個に絞る必要あり。
3. **友達が取ってしまった場合のリカバリー** — 完全に取られたらゲームオーバーに近い状態。どこまで挽回できるか。
